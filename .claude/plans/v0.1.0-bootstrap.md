## High-level architecture

**Plugin** (C#, `net9.0`, Jellyfin 10.11 ABI):
- `BasePlugin<PluginConfiguration>` + `IHasWebPages` for settings
- `IPluginServiceRegistrator` registers a singleton `Debouncer`, singleton `MovieCatalogBuilder`, singleton `GitHubPusher`, named `HttpClient "github"`, and a hosted `LibraryEventListener`
- Library events `ItemAdded` / `ItemUpdated` / `ItemRemoved` filter `item is Movie`, call `Debouncer.Bump()`. Handler returns immediately (no async in handler body) - actual build+push runs on background `Task`
- Debouncer uses a 30s quiet window (configurable). Critical because Jellyfin 10.11.0+ has known `ItemAdded` false-fires on metadata save with NFO savers (jellyfin#15245)
- `GitHubPusher` calls Contents API: GET-current-SHA then PUT, retry once on 409 with refreshed SHA, hard-fail on 401/403/422
- `IScheduledTask` "Resync Movie Catalog Now" (manual trigger only, no recurring) as safety net
- Logging via `ILoggerFactory.CreateLogger("JellyfinMovieCatalog")` - never `ILogger<T>` (known DI failures across 10.8-10.11)

**Viewer** (static, GitHub Pages from `main` branch, `/docs` path):
- `index.html` + `app.js` + `style.css` + `movies.json` (plugin overwrites)
- Vanilla JS, ~50 LOC. `<table>` + `<input type="search">`. Click column headers to sort. Filter is a substring match on title.

**Per-movie JSON schema**:
```json
{
  "title": "string",
  "year": 1234,
  "runtime_seconds": 6420,
  "genres": ["Action", "Thriller"],
  "date_added": "2026-05-22T14:32:11.0000000Z",
  "tmdb_id": "603"
}
```
`year`, `runtime_seconds`, `tmdb_id` are `null` when missing. `date_added` is UTC ISO 8601 (`DateTime.ToUniversalTime().ToString("O")`).

## Debouncer contract

```csharp
class Debouncer
{
  readonly object _gate = new();
  CancellationTokenSource? _wait;
  Task? _runningPush;
  bool _dirtyDuringPush;
  readonly TimeSpan _quietWindow;

  public void Bump()
  {
    lock (_gate)
    {
      if (_runningPush != null) { _dirtyDuringPush = true; return; }
      _wait?.Cancel();
      _wait = new CancellationTokenSource();
      _ = WaitThenFire(_wait.Token);
    }
  }

  async Task WaitThenFire(CancellationToken ct)
  {
    try { await Task.Delay(_quietWindow, ct); }
    catch (TaskCanceledException) { return; }
    await FireAsync();
  }

  async Task FireAsync()
  {
    Task me;
    lock (_gate)
    {
      if (_runningPush != null) { _dirtyDuringPush = true; return; }
      me = _runningPush = Task.Run(DoBuildAndPush);
    }
    try { await me; }
    finally
    {
      lock (_gate)
      {
        _runningPush = null;
        var requeue = _dirtyDuringPush;
        _dirtyDuringPush = false;
        if (requeue) Bump();
      }
    }
  }

  public Task FlushAsync()
  {
    lock (_gate) { _wait?.Cancel(); }
    return _runningPush ?? FireAsync();
  }
}
```

Thread-safety: every state mutation under `_gate`; never await inside the lock. `_runningPush` is the in-flight flag; `_dirtyDuringPush` collapses N concurrent events into one follow-up fire.

## GitHub push contract

Endpoint: `https://api.github.com`. Headers: `Authorization: Bearer <PAT>`, `Accept: application/vnd.github+json`, `X-GitHub-Api-Version: 2022-11-28`, `User-Agent: jellyfin-movie-catalog/<plugin-version>`.

```
PushAsync(byte[] payload, ct):
  for attempt in 1..2:
    sha = null
    get = GET /repos/{owner}/{repo}/contents/{path}?ref={branch}
    switch get.status:
      200: sha = get.json["sha"]
      404: sha = null               // first push
      401|403: raise AuthError      // PAT bad or scope missing
      other: raise TransientError

    body = {
      message: $"movie catalog sync {DateTime.UtcNow:o}",
      content: base64(payload),
      branch: branch,
      sha: sha    // omit when null
    }
    put = PUT /repos/{owner}/{repo}/contents/{path}
    switch put.status:
      200|201: log "pushed N movies, commit=<sha>"; return
      409: continue                // SHA stale, refetch and retry once
      422: log + raise             // malformed; do not retry
      401|403: raise AuthError
      429: honor Retry-After; raise if attempt == 2

  raise StaleConflictError
```

PAT scopes: fine-grained PAT with `Contents: read+write` on the single repo (preferred), or classic PAT with `repo` scope. Document on the settings page.

## Settings page UX (configPage.html)

User enters the PAT at runtime via Jellyfin's admin Dashboard → Plugins → Movie Catalog. Standard Jellyfin plugin config page pattern (`IHasWebPages` + embedded HTML resource + `ApiClient.getPluginConfiguration` / `updatePluginConfiguration`). Concrete form requirements:

| Field | Input | Notes |
|---|---|---|
| Owner | `<input type="text">` | GitHub username/org |
| Repo | `<input type="text">` | Target repo name |
| Branch | `<input type="text">` default `main` | |
| JsonPath | `<input type="text">` default `docs/movies.json` | Path inside the repo where the JSON lands |
| PatToken | `<input type="password">` with a "Show" eye toggle button | Masked by default |
| DebounceSeconds | `<input type="number">` default 30, min 5 | Quiet-window length |

Two buttons:

- **Test Connection**: client-side JS reads the unsaved form values, calls `GET https://api.github.com/repos/{owner}/{repo}/contents/{path}` with the entered PAT, reports one of:
  - `OK - file exists, current SHA: abc1234...` (200)
  - `OK - file does not yet exist; first sync will create it` (404)
  - `Auth failed - check your PAT` (401/403)
  - `Repo not found - check owner/repo` (also 404 if the repo itself is missing - distinguished by checking `/repos/{owner}/{repo}` separately)
- **Save**: standard `ApiClient.updatePluginConfiguration` round-trip. Page reloads to confirm persistence.

Inline help text below the PAT field, verbatim:

> The PAT is stored unencrypted in Jellyfin's plugin config directory on disk. Use a [fine-grained personal access token](https://github.com/settings/personal-access-tokens/new) scoped to **only this one repository** with `Contents: read+write` permission. Blast radius if Jellyfin is ever compromised is bounded to this repo. Rotate the PAT here at any time; the next sync uses the new value immediately.

## Viewer JS shape

```js
const $ = (s) => document.querySelector(s);
let rows = [];
let sortKey = "title", sortAsc = true;

fetch("movies.json")
  .then((r) => r.json())
  .then((data) => { rows = data.movies; render(); });

$("#q").addEventListener("input", render);
document.querySelectorAll("th[data-key]").forEach((th) =>
  th.addEventListener("click", () => {
    const k = th.dataset.key;
    sortAsc = sortKey === k ? !sortAsc : true;
    sortKey = k;
    render();
  }));

function render() {
  const q = $("#q").value.toLowerCase();
  const filtered = rows
    .filter((r) => r.title.toLowerCase().includes(q))
    .sort((a, b) => {
      const av = a[sortKey], bv = b[sortKey];
      if (av == null) return 1;
      if (bv == null) return -1;
      return (av > bv ? 1 : av < bv ? -1 : 0) * (sortAsc ? 1 : -1);
    });
  $("#tbody").innerHTML = filtered.map((r) => `
    <tr><td>${esc(r.title)}</td><td>${r.year ?? ""}</td>
    <td>${fmtRuntime(r.runtime_seconds)}</td>
    <td>${(r.genres ?? []).map(esc).join(", ")}</td>
    <td>${r.date_added?.slice(0,10) ?? ""}</td></tr>`).join("");
  $("#count").textContent = `${filtered.length} of ${rows.length}`;
}

function esc(s) { return String(s).replace(/[&<>]/g, (c) => ({ "&": "&amp;", "<": "&lt;", ">": "&gt;" }[c])); }
function fmtRuntime(s) { if (!s) return ""; const m = Math.round(s / 60); return `${Math.floor(m / 60)}h ${m % 60}m`; }
```

## Risks and watchouts

1. **Plugin GUID**: Template ships with a placeholder GUID. Generate fresh, freeze it. Multiple plugins from the same template GUID will collide.
2. **Async in event handlers**: `ILibraryManager` events run on the scan thread. Handler must return immediately; Debouncer.Bump() is fire-and-forget; actual build+push runs on background Task.
3. **`GetItemList` performance**: Fine for hundreds of movies; for 10k+ consider `EnableTotalRecordCount = false` and don't force metadata refresh.
4. **GitHub Contents API 1MB limit**: Our schema fits ~2-5k movies easily. Log warning at >900KB payload; >10k movies needs the git data API (blob → tree → commit).
5. **PAT plaintext on disk**: Jellyfin stores plugin config unencrypted under `/config/plugins/configurations/`. Document on settings page. Recommend fine-grained PAT scoped to one repo. Never log it.
6. **`ItemUpdated` fires for played/unplayed flips**: Debouncer absorbs them; if we ever care, hash the projected movie JSON and skip pushes where nothing relevant changed.
7. **Time zone**: Force `DateTime.ToUniversalTime().ToString("O")` for `date_added`. Unraid local time is irrelevant.
8. **Unraid Docker volume layout**: `/config` in container maps to a user share like `/mnt/user/appdata/jellyfin/`. Document exact host path for the deploy steps.
9. **Manifest checksum**: MD5 of zip; mismatch = silent install failure. CI computes and injects.
10. **Jellyfin 10.11 → 10.12**: Plugin API isn't stable across minors. `meta.json` declares `targetAbi: 10.11.0.0`. Re-evaluate when 10.12 lands.
11. **gh CLI keyring timeout**: Env check flagged a keyring timeout on `gh auth status`. Token retrieval still works (confirmed by `gh repo view ghill11/apex-platform` succeeding), but keep an eye out - if `gh repo create` or `gh api` fails with auth errors at step 2 or 4a, fall back to user creating the repo via web UI and us pushing via plain git over HTTPS.
12. **First-boot Jellyfin wizard**: Local dev container needs the wizard skipped or scripted, otherwise every restart re-prompts. Scripting via `POST /Startup/Configuration` works.

## Lifecycle

Notes here are intentionally durable. They graduate (in part or in whole) into formal rule files (`.claude/rules/*.md`) when those rule files get drafted in earnest. Once a rule file absorbs the content, the corresponding note either stays as deeper context or gets removed from this catalog - judged at graduation time.

Convention: files starting with `feedback_` capture user-feedback memory items (terse). Files starting with `project_` capture project-specific operational facts. Files starting with `reference_` capture reusable how-to references. Files under `incidents/` capture specific incidents with the symptom/cause/proper-solution structure (see `.claude/agents/incident-recorder.md`). Files under `deploys/` capture per-environment deploy verdict logs (see `.claude/agents/smoke-tester.md`).
